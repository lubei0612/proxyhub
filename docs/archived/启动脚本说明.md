# 🚀 启动脚本说明

## 📋 可用的启动脚本

### 1. 本地开发-启动.bat ⭐⭐⭐⭐⭐ **强烈推荐**

**用途**：本地开发环境，速度快，调试方便

**使用场景**：
- 日常开发和调试
- 快速测试功能
- 不受Docker网络限制

**启动步骤**：
1. 双击运行 `本地开发-启动.bat`
2. 等待数据库启动和后端配置完成
3. 按照提示，打开两个新的PowerShell窗口
4. 窗口1运行：`cd backend && npm run start:dev`
5. 窗口2运行：`cd frontend && npm install && npm run dev`
6. 访问：http://localhost:5173

**优点**：
- ✅ 启动快（约30秒）
- ✅ 热重载，代码修改自动刷新
- ✅ 调试方便
- ✅ 不受Docker网络限制

---

### 2. 启动本地测试.bat ⭐⭐⭐

**用途**：Docker生产环境测试

**使用场景**：
- 测试生产环境配置
- 验收测试
- Docker镜像构建测试

**前提条件**：
- 已配置Docker镜像加速器
- Docker网络连接正常

**启动步骤**：
1. 先运行 `配置Docker镜像加速器.bat` 配置镜像加速
2. 双击运行 `启动本地测试.bat`
3. 等待约5-10分钟（首次构建需要下载依赖）
4. 访问：http://localhost

---

### 3. 快速修复-重新构建.bat ⭐⭐

**用途**：Docker环境出问题时重新构建

**使用场景**：
- Docker容器代码没更新
- Docker服务启动失败
- 需要清理旧的镜像和容器

**注意**：
- 会删除所有数据
- 需要重新初始化测试数据
- 首次构建需要5-10分钟

---

### 4. 停止本地测试.bat

**用途**：停止所有Docker服务

---

### 5. 清理测试数据.bat

**用途**：清理扩展测试数据，保留基础数据

**使用场景**：
- 腾讯云部署前清理测试数据
- 重置数据库到初始状态

---

### 6. 配置Docker镜像加速器.bat

**用途**：配置Docker国内镜像加速器

**使用场景**：
- Docker无法连接Docker Hub
- 首次使用Docker环境

---

## 🎯 快速开始

### 首次使用（本地开发）

```powershell
# 1. 启动数据库和配置
.\本地开发-启动.bat

# 2. 新PowerShell窗口 - 启动后端
cd backend
npm run start:dev

# 3. 新PowerShell窗口 - 启动前端
cd frontend
npm install
npm run dev

# 4. 访问应用
# http://localhost:5173
```

### 首次使用（Docker环境）

```powershell
# 1. 配置Docker镜像加速器
.\配置Docker镜像加速器.bat
# 按照提示配置Docker Settings

# 2. 启动测试环境
.\启动本地测试.bat

# 3. 访问应用
# http://localhost
```

---

**建议：优先使用本地开发环境，速度快且不受网络限制！** 🚀
