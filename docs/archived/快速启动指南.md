# ProxyHub 快速启动指南

**更新时间**: 2025-11-03 15:40  
**适用于**: PowerShell用户

---

## 🚀 3种启动方式（任选其一）

### 方式1: PowerShell脚本 ⭐ 推荐

**步骤**:
1. 右键点击 `启动服务.ps1`
2. 选择"使用PowerShell运行"
3. 或者在PowerShell中执行:
```powershell
.\启动服务.ps1
```

**如果报错"无法加载文件"**:
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

然后重新运行脚本。

---

### 方式2: BAT批处理文件

**步骤**:
1. 双击 `启动ProxyHub-完整版.bat`
2. 等待服务启动

**优点**: 无需配置，直接运行

---

### 方式3: 手动启动（最可靠）

**步骤1 - 打开第一个PowerShell窗口**:
```powershell
cd D:\Users\Desktop\proxyhub\backend
npm run start:dev
```

**等待看到**:
```
[Nest] Application successfully started
```

**步骤2 - 打开第二个PowerShell窗口**:
```powershell
cd D:\Users\Desktop\proxyhub\frontend
npm run dev
```

**等待看到**:
```
Local: http://localhost:8080/
```

## ❌ 常见问题

### 问题1: PowerShell报错"&&不是有效语句分隔符"

**原因**: PowerShell不支持`&&`语法

**解决**: 
- 使用方式1的 `启动服务.ps1`
- 或使用方式2的 `.bat` 文件
- 或使用方式3手动启动

---

### 问题2: 端口被占用

**检查命令**:
```powershell
netstat -ano | findstr :8080
netstat -ano | findstr :3000
```

**解决方案**:
```powershell
taskkill /PID <进程ID> /F
```

---

### 问题3: 数据库连接失败

**检查PostgreSQL**:
```powershell
netstat -ano | findstr :5432
```

**检查Redis**:
```powershell
netstat -ano | findstr :6379
```

**解决**: 启动对应的数据库服务
