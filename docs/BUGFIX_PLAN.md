# ProxyHub Bug修复与优化方案

## 问题清单

### 🔴 P0 - 严重问题（功能无法使用）
1. **动态住宅选购显示"开发中"** - 应该正常显示购买界面
2. **静态住宅购买无法添加IP** - 购买流程无法测试
3. **账户中心点击无反应** - 关键功能不可用
4. **个人资料显示404** - 路由错误
5. **无法退出登录** - 严重的用户体验问题

### 🟠 P1 - 高优先级（功能不完善）
6. **动态住宅管理与985proxy不一样** - UI需要重构
7. **筛选功能无法使用**：
   - 订单管理筛选
   - 交易明细筛选
   - 结算记录筛选
   - 充值订单筛选
8. **通知管理显示"开发中"** - 需要完整实现

### 🟡 P2 - 中优先级（UI/UX优化）
9. **钱包充值文字超出容器** - 布局问题
10. **欢迎回来没有显示用户名** - 个性化体验
11. **价格覆盖管理UI不友好** - 需要改为国家卡片式管理

---

## 修复方案

### Phase 1: 路由和基础功能修复 (P0)

#### Task 1.1: 修复账户中心和个人资料路由
**问题**: 账户中心无反应，个人资料404
**方案**:
- 检查路由配置
- 确保`/account/center`和`/account/profile`路由正确
- 修复点击事件绑定

#### Task 1.2: 修复退出登录功能
**问题**: 点击退出无反应
**方案**:
- 检查`logout`方法是否正确调用
- 确保清除token和用户状态
- 重定向到登录页

#### Task 1.3: 修复动态住宅选购页面
**问题**: 显示"开发中"
**方案**:
- 检查`DynamicBuy.vue`组件
- 实现套餐选择和购买流程
- 对接985Proxy API

#### Task 1.4: 修复静态住宅购买流程
**问题**: 无法添加IP
**方案**:
- 检查购物车逻辑
- 修复IP选择和添加功能
- 确保支付流程完整

---

### Phase 2: UI重构 (P1)

#### Task 2.1: 重构动态住宅管理页面
**目标**: 对标985proxy UI
**方案**:
- 添加通道管理功能
- 实现状态切换（运行中/暂停）
- 优化布局和交互

#### Task 2.2: 实现所有筛选功能
**方案**:
- 订单管理：按订单号、类型、状态、时间筛选
- 交易明细：按类型、时间筛选
- 结算记录：按时间筛选
- 充值订单：按订单号、状态、支付方式、金额、时间筛选

#### Task 2.3: 实现通知管理功能
**方案**:
- 创建通知列表组件
- 实现标记已读/未读
- 添加通知分类

---

### Phase 3: UI/UX优化 (P2)

#### Task 3.1: 修复钱包充值布局
**问题**: 文字超出容器
**方案**:
- 调整支付方式选项的padding和margin
- 使用flexbox优化布局
- 确保文字不换行或截断

#### Task 3.2: 添加用户名显示
**方案**:
- 在"欢迎回来"前添加用户昵称
- 格式：`欢迎回来，{nickname}！`

#### Task 3.3: 重构价格覆盖管理
**目标**: 国家卡片式管理
**方案**:
- 显示所有国家的商品卡片
- 点击卡片弹出价格编辑对话框
- 支持批量管理
