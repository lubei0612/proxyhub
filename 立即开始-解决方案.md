# 🚀 立即开始 - 解决Docker网络问题

## ⚠️ 您遇到的问题

Docker无法连接Docker Hub下载镜像：
```
dial tcp 157.240.17.36:443: connectex: A connection attempt failed
```

---

## ✅ 最佳解决方案：使用本地开发环境

**不使用Docker，直接本地运行！** 启动快、调试方便、不受网络限制！

---

## 🎯 5步快速开始（10分钟）

### 步骤1：启动数据库（2分钟）

```powershell
# 只启动PostgreSQL和Redis
docker-compose up postgres redis -d

# 等待10秒
timeout /t 10
```

### 步骤2：配置并启动后端（3分钟）

```powershell
# 进入后端目录
cd backend

# 检查.env文件
if (!(Test-Path .env)) { Copy-Item ..\docs\ENV_TEMPLATE.txt .env }

# 安装依赖（如果还没安装）
npm install

# 初始化测试数据
npm run seed

# 启动后端
npm run start:dev
```

**等待看到**：
```
[Nest] Application successfully started
[Nest] Listening on port 3000
```

### 步骤3：启动前端（新PowerShell窗口，3分钟）

```powershell
# 进入前端目录
cd frontend

# 安装依赖（如果还没安装）
npm install

# 启动前端
npm run dev
```

**等待看到**：
```
➜  Local:   http://localhost:5173/
```

### 步骤4：访问应用

打开浏览器：**http://localhost:5173**

### 步骤5：登录测试

```
邮箱：user@example.com
密码：password123
```

## 🔧 如果遇到问题

### 问题1：端口被占用

```powershell
# 查找占用3000端口的进程
netstat -ano | findstr :3000

# 结束进程
taskkill /F /PID <进程ID>
```

### 问题2：数据库连接失败

```powershell
# 检查数据库状态
docker-compose ps postgres

# 如果不正常，重启
docker-compose restart postgres
```

### 问题3：npm install失败

```powershell
# 使用国内镜像
npm config set registry https://registry.npmmirror.com
npm install
```
